# QuickDesk 信令服务器部署文档

## 系统要求

- CentOS 7/8/Stream 或 Rocky Linux 8+
- 2GB+ 内存
- 10GB+ 磁盘空间
- 公网 IP 和域名（可选）

## 1. 安装 Docker

```bash
# 安装 Docker
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
docker --version
```

## 2. 部署数据库（Docker）

```bash
# 创建数据目录
sudo mkdir -p /data/quickdesk/{postgres,redis}

# 启动 PostgreSQL
docker run -d --name quickdesk-postgres \
  --restart=always \
  -p 5432:5432 \
  -e POSTGRES_USER=quickdesk \
  -e POSTGRES_PASSWORD=quickdesk123 \
  -e POSTGRES_DB=quickdesk \
  -v /data/quickdesk/postgres:/var/lib/postgresql/data \
  postgres:15

# 启动 Redis
docker run -d --name quickdesk-redis \
  --restart=always \
  -p 6379:6379 \
  -v /data/quickdesk/redis:/data \
  redis:7 redis-server --appendonly yes

# 验证运行状态
docker ps
```

## 3. 安装 Go 环境

```bash
# 下载 Go 1.21
cd /tmp
wget https://go.dev/dl/go1.21.6.linux-amd64.tar.gz

# 解压安装
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf go1.21.6.linux-amd64.tar.gz

# 配置环境变量
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
echo 'export GOPROXY=https://goproxy.cn,direct' >> ~/.bashrc
source ~/.bashrc

# 验证安装
go version
```

## 4. 安装 Node.js（构建管理后台前端）

```bash

curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs

# 验证安装
node --version
```

## 5. 编译信令服务器

> **注意：** 数据库表结构由信令服务器启动时 GORM AutoMigrate 自动创建/更新，无需手动执行 SQL。
> 参考 SQL 见 `SignalingServer/migrations/001_init.sql`。

```bash
# 克隆或上传代码到服务器
# 假设代码位于 /opt/quickdesk/SignalingServer

cd /opt/quickdesk/SignalingServer

# 构建管理后台前端（Vue 3 + Element Plus）
cd web
npm install
npm run build
cd ..

# 下载 Go 依赖
go mod tidy

# 编译（前端构建产物会通过 go:embed 嵌入二进制）
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
  go build -a -ldflags="-s -w -extldflags '-static'" \
  -o quickdesk_signaling ./cmd/signaling

# 创建运行目录
sudo mkdir -p /opt/quickdesk/signaling
sudo cp quickdesk_signaling /opt/quickdesk/signaling/
```

## 6. 配置服务

```bash
# 创建配置文件
sudo cat > /opt/quickdesk/signaling/.env << 'EOF'
SERVER_HOST=0.0.0.0
SERVER_PORT=8000
DB_HOST=localhost
DB_PORT=5432
DB_USER=quickdesk
DB_PASSWORD=quickdesk123
DB_NAME=quickdesk
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=

# API Key 客户端认证（可选）
# 设置后，只有携带正确 API Key 的客户端才能连接此服务器。
# 留空表示不启用 API Key 校验（任何客户端都可以连接）。
# 客户端通过 X-API-Key 请求头传递此值。
API_KEY=

# WebClient Origin 域名白名单（可选，多个用逗号分隔）
# 浏览器会自动发送 Origin 请求头且 JS 无法伪造。
# 设置后，只有从这些域名加载的 WebClient 才能访问信令服务器。
# 原生客户端（Qt/Host/Client）使用 API Key 认证，WebClient 使用 Origin 白名单认证。
# 示例: https://web.quickdesk.cc,https://quickdesk.example.com
ALLOWED_ORIGINS=
EOF

# 创建 systemd 服务
sudo cat > /etc/systemd/system/quickdesk-signaling.service << 'EOF'
[Unit]
Description=QuickDesk Signaling Server
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/quickdesk/signaling
ExecStart=/opt/quickdesk/signaling/signaling
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

# 启动服务
sudo systemctl daemon-reload
sudo systemctl start quickdesk-signaling
sudo systemctl enable quickdesk-signaling

# 检查服务状态
sudo systemctl status quickdesk-signaling
```

## 8. 域名访问配置（Nginx 反向代理）

```bash
# 安装 Nginx
sudo yum install -y nginx
sudo systemctl start nginx
sudo systemctl enable nginx

# 配置反向代理
sudo cat > /etc/nginx/conf.d/quickdesk.conf << 'EOF'
upstream signaling {
    server 127.0.0.1:8000;
}

server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名

    client_max_body_size 100M;

    location / {
        proxy_pass http://signaling;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }
}
EOF

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

## 9. HTTPS 配置（可选）

```bash
# 安装 Certbot
sudo yum install -y epel-release
sudo yum install -y certbot python3-certbot-nginx

# 申请证书（自动配置 Nginx）
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run

# Certbot 会自动添加定时任务，无需手动配置
```

配置完成后，Nginx 配置会自动更新为：

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

## 12. 常用运维命令

```bash
# 重启服务
sudo systemctl restart quickdesk-signaling
sudo systemctl restart nginx

# 停止服务
sudo systemctl stop quickdesk-signaling

# 查看数据库连接
docker exec -it quickdesk-postgres psql -U quickdesk -d quickdesk

# 查询设备数量
docker exec -it quickdesk-postgres psql -U quickdesk -d quickdesk -c "SELECT COUNT(*) FROM devices;"

# 备份数据库
docker exec quickdesk-postgres pg_dump -U quickdesk quickdesk > /backup/quickdesk_$(date +%Y%m%d).sql

# 恢复数据库
cat backup.sql | docker exec -i quickdesk-postgres psql -U quickdesk -d quickdesk

# 查看预设配置
curl http://localhost:8000/api/v1/admin/preset

# 更新预设配置（也可通过管理后台 /admin/ 页面操作）
curl -X PUT http://localhost:8000/api/v1/admin/preset \
  -H "Content-Type: application/json" \
  -d @test_preset.json
```

## 故障排查

### 服务无法启动

```bash
# 查看详细日志
sudo journalctl -u quickdesk-signaling -n 100 --no-pager

# 检查端口占用
sudo lsof -i:8000

# 检查数据库连接
docker exec -it quickdesk-postgres psql -U quickdesk -d quickdesk -c "SELECT 1;"
docker exec -it quickdesk-redis redis-cli ping
```

### Nginx 502 错误

```bash
# 检查信令服务是否运行
sudo systemctl status quickdesk-signaling

# 检查 SELinux 状态（可能阻止 Nginx 连接）
sudo setenforce 0  # 临时关闭
sudo sed -i 's/SELINUX=enforcing/SELINUX=disabled/g' /etc/selinux/config  # 永久关闭

# 或者配置 SELinux 允许 Nginx 网络连接
sudo setsebool -P httpd_can_network_connect 1
```

### 数据库连接失败

```bash
# 检查 Docker 容器状态
docker ps -a

# 重启数据库容器
docker restart quickdesk-postgres quickdesk-redis

# 检查数据库日志
docker logs quickdesk-postgres
```

## 安全建议

1. **修改默认密码**：修改 PostgreSQL 和 Redis 密码
2. **配置防火墙**：仅开放必要端口（80、443）
3. **启用 HTTPS**：生产环境必须使用 HTTPS
4. **定期备份**：设置定时任务备份数据库
5. **日志监控**：配置日志收集和告警
6. **限流配置**：在 Nginx 中配置请求限流
7. **客户端认证**：设置 `API_KEY` 限制原生客户端访问，设置 `ALLOWED_ORIGINS` 限制 WebClient 访问

```nginx
# Nginx 限流配置示例
http {
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;
    
    server {
        location /api/ {
            limit_req zone=api_limit burst=20 nodelay;
        }
    }
}
```

```bash
docker run -d --name quickdesk-postgres \

```

- **HTTP**: `http://your-domain.com`

- **WebSocket**: `wss://your-domain.com/signal/:device_id?access_code=xxx`
